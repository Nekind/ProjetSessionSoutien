Compétiteurs de SCM:

Git:

Choississez git, car vous aurez a apprendre git a un moment donné. Git est :
  • + Gratuit
  • + Open-source
  • + Possède des fonctions avancés
  • + Rapide
  • + Le plus populaire au point de dire que c'est un quasi monopolie.
  • - Pas intuitif a apprendre
  • - Mauvais pour la gestion des dépendences
  • - Pas de controle d'acces par défaut, il faut ajouter
  • - Copie les fichiers au complet
  • - Devient lent quand l'historique deviens tres long (+10k commits)

    • Mercurial:

  • Plus facile a apprendre
  • Plus "User Friendly"
  • Cache les fonctions avancés
  • Beaucoup moins populaire

    • Perforce:

  • Extremement rare
  • Pour les logiciels avec des gros fichiers binaires comme des jeux vidéos


    • Fournisseurs d'hébergement:

    Github:

  • Plus populaire
  • Gratuit pour les dossiers privés
  • Tres connu puisque tous les projet open-source sont sur Github
  • Bon pour se trouver un emploi
  • Le forfait GitHub Enterprise commence a $2500 / 10 utilisateurs / année

    • Gitlab:

  • Le code source de Gitlab est open-source
  • Vous pouvez donc hosté vous-meme votre serveur Gitlab
  • Le forfait GitLab Enterprise coute $39 / utilisateur / année

    • Bitbucket:

  • Peux acheter le code source
  • Le forfait Bitbucket Small Teams coute 10$


    • Choisissez Github:
  • Si votre projet est open-source ou pour votre portfolio
  • Github est le plus connu car c'est la qu'il y a le plus de projet visible
    • Choisissez GitLab:
  • Si vous avez de besoin d'une plateforme qui est open source
    • Choisissez BitBucket:
  • Si vous utilisez plusieurs logiciels de Atlassian
